# How to Register Custom SpecialHandler Back Parsers

[简体中文](../zh_cn/custom-special-handler-parser.md) **English**

> Generated by GLM-5.1

## Overview

In HexCasting, certain patterns are handled by `SpecialHandler` rather than a regular `Action`. When HexParse's `PatternParser` performs backward parsing (NBT → String) and encounters a `SpecialHandler` pattern, it looks up registered back parsers to generate a string representation.

Through `HexParseAPI.AddSpecialHandlerBackParser`, you can register backward parsing logic for your custom `SpecialHandler` types.

## SpecialHandler Back Parsing Flow

When `PatternParser` parses a pattern Iota, the execution flow is:

1. Reconstruct the `HexPattern` object from NBT
2. Call `PatternRegistryManifest.matchPattern()` to match the pattern
3. If the match result is `PatternShapeMatch.Special`, retrieve its `SpecialHandler` instance
4. Look up the parsing function for the `SpecialHandler`'s `Class` in `SPECIAL_HANDLER_MAP`
5. If found, call the function to generate a string; otherwise, fall back to the angle signature (`_anglesSignature`)

## API Method

### AddSpecialHandlerBackParser

```kotlin
HexParseAPI.AddSpecialHandlerBackParser(
    cls: Class<T>,
    func: (T, CompoundTag) -> String
)
```

| Parameter | Type | Description |
|-----------|------|-------------|
| `cls` | `Class<T>` | The `Class` object of the `SpecialHandler` subclass |
| `func` | `(T, CompoundTag) -> String` | The back parser function that receives the `SpecialHandler` instance and the raw NBT node, returning a string representation |

- `T` must be a subclass of `SpecialHandler`
- The first parameter of `func` is the matched `SpecialHandler` instance, from which you can directly extract data
- The second parameter of `func` is the full `CompoundTag` of the pattern Iota, useful for accessing additional data

## Built-in SpecialHandler Parsers

HexParse has registered back parsers for the following `SpecialHandler` types:

| SpecialHandler Class | Output Format | Description |
|---------------------|---------------|-------------|
| `SpecialHandlerMask` | `mask_...` | Mask operation; `-` for masked, `v` for visible |
| `SpecialHandlerNumberLiteral` | `num_N` | Number literal, using minimal float representation |

### SpecialHandlerMask Example

For a 3-bit mask (masked, visible, masked), the output is `mask_-v-`.

### SpecialHandlerNumberLiteral Example

For the number `3.14`, the output is `num_3.14`.

## Practical Examples

### Example: Register a Custom SpecialHandler Back Parser

Suppose you have a custom `SpecialHandler` that represents a color value:

```kotlin
import at.petrak.hexcasting.api.casting.castables.SpecialHandler
import io.yukkuric.hexparse.api.HexParseAPI
import net.minecraft.nbt.CompoundTag

class ColorSpecialHandler(val color: Int) : SpecialHandler {
    companion object {
        fun registerBackParser() {
            HexParseAPI.AddSpecialHandlerBackParser(
                ColorSpecialHandler::class.java
            ) { handler, node ->
                "color_%06x".format(handler.color)
            }
        }
    }
}

ColorSpecialHandler.registerBackParser()
```

### Registering in Java

```java
import at.petrak.hexcasting.api.casting.castables.SpecialHandler;
import io.yukkuric.hexparse.api.HexParseAPI;
import net.minecraft.nbt.CompoundTag;

public class MySpecialHandler extends SpecialHandler {
    public static void registerBackParser() {
        HexParseAPI.AddSpecialHandlerBackParser(
            MySpecialHandler.class,
            (handler, node) -> "myhandler_" + handler.getValue()
        );
    }
}
```

## Important Notes

- Registration should be done during the mod initialization phase to ensure it occurs before the first parse
- Each `SpecialHandler` class can only have one back parser registered; re-registering overwrites the previous one
- If no back parser is registered for a `SpecialHandler`, `PatternParser` falls back to outputting the angle signature (`_` prefix + angle string), which still works but is less readable
- You can use `INbt2Str.displayMinimalStatic(Double)` within `func` as a helper method to format floating-point numbers
