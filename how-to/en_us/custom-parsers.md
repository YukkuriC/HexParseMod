# How to Register Custom Parsers

[简体中文](../zh_cn/custom-parsers.md) **English**

> Generated by GLM-5.1

## Overview

HexParse provides a bidirectional parsing pipeline: **forward parsing** (String → NBT) and **backward parsing** (NBT → String). Through `HexParseAPI`'s `AddForthParser` and `AddBackParser`, you can register custom parsers to extend the Iota types or syntax formats that HexParse supports.

Parsers are matched in registration order — the first parser whose `match` returns `true` will handle the node.

## Key Interfaces

### IStr2Nbt — Forward Parser (String → NBT)

```java
public interface IStr2Nbt {
    boolean match(String node);
    CompoundTag parse(String node);
    default boolean ignored() { return false; }
    default int getCost() { return HexParseConfig.parserBaseCost(); }
}
```

| Method | Description |
|--------|-------------|
| `match(String node)` | Checks whether the input string fragment can be handled by this parser |
| `parse(String node)` | Parses the matched string into a `CompoundTag` (the NBT-serialized form of an Iota) |
| `ignored()` | If `true`, the parse result is discarded (e.g., for pure comments/indents) |
| `getCost()` | Returns the media cost for parsing this node; defaults to the `parserBaseCost` config value |

### INbt2Str — Backward Parser (NBT → String)

```java
public interface INbt2Str extends IConfigNumReceiver {
    boolean match(CompoundTag node);
    String parse(CompoundTag node);
    default boolean isType(CompoundTag node, String type) { ... }
    default String displayMinimal(Double raw) { ... }
}
```

| Method | Description |
|--------|-------------|
| `match(CompoundTag node)` | Checks whether the NBT node can be handled by this parser |
| `parse(CompoundTag node)` | Converts the matched NBT node to its string representation |
| `isType(CompoundTag node, String type)` | Helper method that checks whether the node's `type` field matches |
| `displayMinimal(Double raw)` | Helper method that formats a floating-point number to its minimal representation (trailing zeros removed) |

### IPlayerBinder — Player Binding Interface

If your parser needs access to player information (e.g., for entity references or permission checks), implement this interface:

```java
public interface IPlayerBinder {
    void BindPlayer(ServerPlayer p);
}
```

Before each parse call, `ParserMain` automatically calls `BindPlayer` on all parsers implementing `IPlayerBinder`, passing in the current casting player.

### IConfigNumReceiver — Config Number Interface

`INbt2Str` extends this interface for receiving parsing config bitmasks:

```java
public interface IConfigNumReceiver {
    default void receiveConfigNum(int configNum) {}
    default boolean hasConfigNum(int mine, int comparer) {
        return (mine & comparer) == comparer;
    }
}
```

## API Methods

### AddForthParser

```kotlin
HexParseAPI.AddForthParser(p: IStr2Nbt)
```

Registers a forward parser. The parser is appended to the end of the parser list and is tried after all built-in parsers.

### AddBackParser

```kotlin
HexParseAPI.AddBackParser(p: INbt2Str)
```

Registers a backward parser. The parser is appended to the end of the parser list and is tried after all built-in parsers.

## Practical Examples

### Example 1: Register a Simple Forward Parser

The following example creates a parser that converts strings with the `mytype_` prefix into custom Iota NBT:

```kotlin
import io.yukkuric.hexparse.api.HexParseAPI
import io.yukkuric.hexparse.parsers.str2nbt.BaseConstParser
import net.minecraft.nbt.CompoundTag

class MyTypeParser : BaseConstParser.Prefix("mytype_") {
    override fun parse(node: String): CompoundTag {
        val value = node.substring(8)
        val tag = CompoundTag()
        tag.putString("type", "mymod:my_type")
        tag.putString("data", value)
        return tag
    }
}

HexParseAPI.AddForthParser(MyTypeParser())
```

`BaseConstParser.Prefix` is a built-in convenience base class that automatically implements `match` based on the provided prefix string.

### Example 2: Register a Backward Parser

```kotlin
import io.yukkuric.hexparse.api.HexParseAPI
import io.yukkuric.hexparse.parsers.nbt2str.INbt2Str
import net.minecraft.nbt.CompoundTag

class MyTypeBackParser : INbt2Str {
    companion object {
        const val TYPE_ID = "mymod:my_type"
    }

    override fun match(node: CompoundTag): Boolean {
        return isType(node, TYPE_ID)
    }

    override fun parse(node: CompoundTag): String {
        val data = node.getString("data")
        return "mytype_$data"
    }
}

HexParseAPI.AddBackParser(MyTypeBackParser())
```

### Example 3: Register an Ignored Parser

Some parsers produce Iotas that have no effect during actual casting (e.g., comments). Return `true` from `ignored()` to skip them:

```kotlin
import io.yukkuric.hexparse.api.HexParseAPI
import io.yukkuric.hexparse.parsers.str2nbt.IStr2Nbt
import net.minecraft.nbt.CompoundTag

class MyIgnoredParser : IStr2Nbt {
    override fun match(node: String): Boolean = node.startsWith("ignore_")

    override fun parse(node: String): CompoundTag {
        // Return value doesn't matter since ignored() returns true
        return CompoundTag()
    }

    override fun ignored(): Boolean = true
}

HexParseAPI.AddForthParser(MyIgnoredParser())
```

### Example 4: Custom Parse Cost

```kotlin
import io.yukkuric.hexparse.api.HexParseAPI
import io.yukkuric.hexparse.parsers.str2nbt.BaseConstParser
import net.minecraft.nbt.CompoundTag

class MyExpensiveParser : BaseConstParser.Prefix("expensive_") {
    override fun parse(node: String): CompoundTag {
        // Advanced parsing logic...
        return CompoundTag()
    }

    override fun getCost(): Int {
        return 1000  // Custom media cost
    }
}

HexParseAPI.AddForthParser(MyExpensiveParser())
```

## Built-in Parser Reference

HexParse includes the following forward parsers (in match order):

| Parser | Prefix/Match Rule | Description |
|--------|-------------------|-------------|
| `ToPattern.META` | Meta op names (`(`, `)`, `\`, `del`, etc.) | Meta operation patterns |
| `ToMiscConst` | `self`, `true`, `false`, `null`, `garbage` | Constants |
| `FallbackBinaryParser.STR2NBT` | `nbt_` | Binary NBT fallback |
| `ToPattern.NORMAL` | Pattern names | Normal patterns |
| `ToPattern.GREAT` | Great spell names | Great patterns (requires unlock) |
| `Comment.Indent` | `tab_N` | Indentation |
| `Comment` | `comment_` / `c"..."` | Comments |
| `ToNum` | Numbers | Numeric Iota |
| `ToVec` | `vec_X_Y_Z` | Vector Iota |
| `ToMask` | `mask_...` | Mask special handler |
| `ToNumPattern` | `num_N` | Number literal special handler |
| `ToEntity` | `entity_<UUID>` | Entity reference |
| `ToDialect` | Dialect mapping | Custom name mapping |

Built-in backward parsers (in match order):

| Parser | Matched Type | Output Format |
|--------|-------------|---------------|
| `PatternParser` | `hexcasting:pattern` | Pattern name / angle signature |
| `CommentParser` | `hexparse:comment` | `comment_...` / `/*...*/` |
| `NumParser` | `hexcasting:double` | Minimal float |
| `VecParser` | `hexcasting:vec3` | `vec_X_Y_Z` |
| `EntityParser` | `hexcasting:entity` | `self` / `entity_<UUID>` |
| `BoolParser` | `hexcasting:boolean` | `true` / `false` |
| `NullParser` | `hexcasting:null` | `null` |
| `GarbageParser` | `hexcasting:garbage` | `garbage` |

## Important Notes

- Custom parsers are appended to the end of the list, so they have **lower priority than all built-in parsers**. If your custom prefix conflicts with a built-in parser, the built-in one will match first
- For higher priority, you would need to register before `ParserMain.init()`, or use naming that doesn't conflict with built-in prefixes
- `IPlayerBinder.BindPlayer` is called before every parse invocation — do not store long-term state in it
- The cost returned by `getCost()` is accumulated via `CostTracker` during parsing and ultimately deducted from the caster
